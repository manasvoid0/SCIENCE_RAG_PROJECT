2The two AI models

They do different jobs and cannot replace each other.

🧠

qwen2.5 the brain

Reads, reasons, and writes. It writes the tutor's answers and judges whether an essay looks AI-written.

Use qwen2.5:3b on 8 GB RAM · 7b on 16 GB+

📏

nomic-embed-text the meaning-ruler

Turns text into numbers that capture meaning. Used to search the textbook and to compare meaning between students.

Tiny & fast — never the weak link.

nomic finds & compares · qwen thinks & writes

3Science Tutor — how a question is answered (RAG)

RAG = Retrieval-Augmented Generation. We fetch real textbook text first, then let the AI answer from it — so it can't make things up.

Youtype a question
→
nomic-embedquestion → numbers
→
FAISSfind 3 closest chunks
→
qwen2.5writes the answer
→
Yousee the answer

Setup happens once in app.py: read the PDF → chop into chunks → turn into numbers → save to the FAISS index.

4Plagiarism Detector — from upload to report

1
Student uploads a file → saved into submissions/. No checking yet.
2
Teacher clicks "Run analysis". Old reports are cleared first.
3
Extract text from each PDF / Word / PPT / TXT.
4
Run the 4 detectors (see next section).
5
Combine the scores into one integrity score (0–100) per student.
6
Save a report for each student into reports/.
7
Teacher sees a colour-coded table + per-student detail with flagged lines.
8
The teacher decides — the system only scores and flags.

5The detection checks

Two checks measure (exact, model-size doesn't matter) and one judges (needs the brain).

Copy check TF-IDF · math

Catches student-vs-student copy-paste by comparing which words were used. Rock-solid & instant.

Paraphrase check embeddings

Catches reworded copying by comparing meaning. Beats simple synonym-swapping.

AI detector qwen2.5 · judges

Reads the essay and judges if it looks AI-written. The only check that needs the thinking model.

Originality positive · math

Rewards own work: vocabulary diversity, sentence-length variation, personal phrases.
